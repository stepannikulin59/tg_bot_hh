# tg_bot_hh

Учебный Telegram-бот для мониторинга вакансий hh.ru.

## Переменные окружения

Обязательные:
- `TELEGRAM_BOT_TOKEN`
- `HH_USER_AGENT`
- `TARGET_AREA_NAME`
- `STATE_PATH`

Необязательные:
- `POLL_INTERVAL_SECONDS` (по умолчанию `300`)
- `HH_REQUEST_LIMIT_PER_CYCLE` (по умолчанию `60`)
- `HTTP_TIMEOUT_SECONDS` (по умолчанию `30`)
- `LOG_LEVEL` (по умолчанию `INFO`)

Пример: [`.env.example`](.env.example).

## Запуск локально

1. Создайте `.env` на основе `.env.example`.
2. Установите зависимости:

```bash
pip install -r requirements.txt
```

3. Запустите бота:

```bash
PYTHONPATH=src python3 -m tg_bot_hh.main
```

## Запуск в Docker

1. Создайте `.env` на основе `.env.example`.
2. Поднимите сервис:

```bash
docker compose up --build
```

Состояние хранится в SQLite-файле в volume `tg_bot_hh_state` по пути `/data/state.db`.

## Остановка Docker

```bash
docker compose down
```

Без флага `-v` volume состояния сохраняется.

## Команды бота

- `/start`: привязывает первый чат, включает автоопрос
  и отправляет топ-10 свежих вакансий списком.
- `/stop`: выключает автоопрос для привязанного чата.

## Текущая логика

- Поиск идет по двум веткам: Пермь и удаленная работа.
- Фильтрация по ключевым словам в названии/описании.
- `/start` всегда повторяет поиск
  и показывает до 10 последних вакансий.
- Автоопрос каждые 5 минут отправляет только новые вакансии.
- Вывод идет пакетами по 10 вакансий
  с заголовком `Вакансии на YYYY-MM-DD HH:MM`.
- При ошибках HH/API бот отправляет уведомление в чат один раз до восстановления.
- После первого успешного цикла отправляется уведомление о восстановлении.
- Токен Telegram маскируется в логах, а `httpx/httpcore` переведены на уровень `WARNING`.
